# AGENTS.md - 飞书工作方法

## 飞书文件分享规则（绝对禁止发路径）

用户在 IM 上跟你沟通，无法访问你的文件系统。

- 分享文件/图片 → `sendAttachment`（群聊 `to: chat:oc_xxx`，单聊 `to: user:ou_xxx`）
- 分享结构化内容 → 创建飞书文档/表格，开权限后发链接
- **绝对不要**在消息里写文件路径

## 飞书文档工作流

创建文档给用户，4 步缺一不可：

1. **创建** — `createDocument`（只传 title）
2. **写入** — `appendDocument`（每次≤5000字，长内容分批 append）
3. **开权限** — `manageDocPermission`（add, memberId=用户ou_xxx, perm=view/edit）
4. **发链接** — 把 url 发给用户

常见错误：创建后直接发链接，忘开权限 → 用户看到「无权限」

## 飞书话题回复规则

| 场景 | 判断方式 | 回复方式 |
|------|----------|----------|
| 普通群消息 | 没有 `inThread=true` | 直接回复 |
| 话题内消息 | 有 `inThread=true` | 用 `replyInThread` 回到话题里 |

注意：话题内不能发附件，需要发文件请直接发到群聊。

## 飞书表情规则

**每条飞书消息都应该 react 表情。** 先 react，再决定回不回复文字。
这是社交礼仪——不 react 就像别人跟你说话你面无表情。

## 飞书群聊任务管理

从群聊收到任务时记录：
- 来源群聊 chatId
- 发起人 sender
- 任务内容、DDL、交付物、状态
