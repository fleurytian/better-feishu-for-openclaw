# AGENTS.md - 工作方法（飞书相关部分模板）

> 将此内容合并到你的 `~/.openclaw/workspace/AGENTS.md`

## 飞书文件分享规则（重要）

**你使用着用户的计算机，但用户是在 IM 上跟你沟通。** 他们无法直接访问你发的相对路径或本地路径。

**正确做法：**
- 通过 `sendAttachment` 发送文件/图片附件
- 通过飞书文档/表格分享结构化内容（记得开权限）
- 发送可访问的 URL 链接

**错误做法：**
- 只回复一句「文件已保存到 /tmp/xxx.png」— 用户打不开
- 发送本地路径期望用户自己找 — 用户看不到你的文件系统

**示例：**
```
❌ 错误：图表已生成，保存在 /tmp/chart.png
✅ 正确：[使用 sendAttachment 发送 /tmp/chart.png]
```

## 飞书群聊任务管理

**任务记录规则（必须）：**
每次从飞书群聊收到任务时，必须记录，并注明：
- 任务来源：群聊 chatId（如 `oc_xxx`）或单聊
- 任务发起人：谁的指令（从消息的 sender 获取）
- 任务内容：具体要求
- DDL（截止时间）：必须写明具体日期时间
- 交付物：预期产出
- 状态：进行中/已完成

示例：
```
### 任务XX：整理会议纪要
- **来源群聊：** oc_xxx（项目群）
- **发起人：** 张三
- **任务内容：** 整理今天下午的会议纪要
- **DDL：** 2026-02-07 18:00
- **交付物：** 飞书文档链接
- **状态：** 进行中
```

## 飞书旁听规则

如果没有 @ 你，消息前面会注明 `[旁听，保持旁听不回复则输出NO_REPLY]`。

**旁听时的行为：**
- 被 @ 时必须回复
- 旁听到感兴趣/相关的话题可以选择插话
- 不想回复时，回复 `NO_REPLY`（不会发送到群里）
- 不要对同一条消息反复回复

**判断是否回复的参考：**
- 话题与我相关吗？
- 我有有价值的信息要补充吗？
- 这是闲聊还是需要帮助？
- 插话会不会打断别人的对话？

## 飞书话题回复规则

| 场景 | 如何判断 | 回复方式 |
|------|----------|----------|
| 普通群消息 | 没有 `inThread=true` | 直接回复即可 |
| 话题内消息 | 有 `inThread=true` | **必须用 `replyInThread` 回复到话题里**，不要用 send 发到主群。**注意：话题内不能发附件**，需要发文件/图片请直接发到群聊 |
| 想主动开启话题 | 需要针对某条消息深入讨论 | 使用 `replyInThread` action |

**示例：**
```json
{ "action": "replyInThread", "messageId": "om_xxx", "content": "这是话题回复内容" }
```

## 飞书表情规则

**每条飞书消息都应该点表情回应。** 这是社交礼仪。

- 先点表情（react），再决定要不要回复文字
- 不点表情就像见面不打招呼

## 飞书文档工作流

创建文档给用户时，必须完成 4 步：

1. **创建文档** — `createDocument`
2. **分批写入** — `appendDocument`（每次≤5000字）
3. **开放权限** — `manageDocPermission`（add, memberId=用户ou_xxx, perm=view/edit）
4. **发送链接** — 把 url 发给用户

**常见错误：** 创建后直接发链接，忘记第3步开权限 → 用户看到「无权限」
